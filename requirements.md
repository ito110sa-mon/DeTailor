# DeTailor 要件定義書

日記の内容を生成・肉付けするWebアプリケーション「DeTailor」の要件定義ドキュメントです。

## 1. プロジェクト概要
日常の些細な「詳細 (Detail)」を、AIが美しい文章に「仕立てる (Tailor)」日記支援アプリ。
ユーザーが入力した短いタイトルや本文（メモ）をもとに、選択したスタイルに応じた豊かな日記本文を生成・肉付けします。

## 2. 主要機能

### A. 仕立て機能 (肉付け・本文生成)
- **日記の基本構成**: 「タイトル」と「本文」
- **生成の入力制限とバリデーションルール**:
  - **タイトル・本文ともに未入力**: 生成不可（バリデーションエラーを表示）。
  - **タイトルと本文の少なくともどちらか一方が入力されている場合**: 本文の肉付け・生成処理を行う。
  - **タイトルのみ入力されている場合**: タイトルから内容を推測し、例外的に本文を丸ごと生成する。

### B. スタイル（仕立て方）の選択
AIが生成する文章のトーン＆マナーを以下の5つのスタイルから選択可能。
1. **エッセイ風**: 日常の気づきや感情を綴った、親しみやすく自然な文章。
2. **小説風**: ドラマチックで情緒的な情景描写を取り入れたストーリー調の文章。
3. **箇条書き風**: 出来事や事実、気づきをシンプルに整理した読みやすい形式。
4. **ブログ風**: 見出しやまとめなどを意識し、他者に伝えることを目的とした構成の文章。
5. **万バズ投稿風**: SNS（X等）で多くのリアクションを獲得しそうな、フックがあり共感を呼ぶトーン。

### C. 詳細なオーダー（カスタム指示）
- **概要**: ユーザーが仕立て（肉付け）の過程において、AIに対して個別の要望（詳細なオーダー）を記述できる機能。
- **指示の例**: 「もっと嬉しそうに書いて」「英語を交えて」「句読点を多めにする」などの自由記述。
- **プロンプトへの反映**: 選択したスタイルよりもこのカスタム指示が優先して適用される。

### D. 推敲・エディタ・コピー機能
- **編集機能**: AIが生成・肉付けしたタイトルと本文を、ユーザーが画面上のエディタで個別に自由に追記・修正可能。
- **ワンクリックコピー機能**: 仕立て上がった「タイトル」と「本文」を、それぞれ個別のボタンからワンクリックでクリップボードにコピーできる。

---

## 3. 非機能要件・技術要件
- **フロントエンド技術**: React + Vite (TypeScript)
- **スタイリング**: Vanilla CSS (高級テーラーをモチーフにした独自のダークシックデザイン、縫製をイメージしたアニメーション)
- **動作環境**: Webブラウザ（PC / スマートフォン両対応のレスポンシブデザイン）
- **AI API**: Gemini API (gemini-3.5-flash)
- **データ・セキュリティ管理**:
  - クライアントサイドでのローカル開発時は、LocalStorage もしくは環境変数 `.env.local` からAPIキーを読み込む。
  - 将来的な一般公開時は、ホスティングサービス（Vercel等）のサーバーレス関数（API Routes）にAPIキーを隠蔽し、安全に通信を行う。
